SqlGetFunctions

Record that allows the user to specify which supported ODBC functions should be enabled or disabled. This can be done by setting specific FunctionId values to true or false. This can be used to disable certain features, such as parameters, when they cause errors for a specific driver.

Example:

SqlGetFunctions = [
    SQL_API_SQLBINDPARAMETER = false
]

This example disables parameter binding. This is useful for when parameterized queries are not supported.

RecordEmpty
SqlColumns

Function that allows the user to modify the table returned by calling SQLColumns. This can be used to work around drivers that return incorrect or deprecated types for column metadata when calling SQLColumns.

Example:

SQLColumns = (catalogName, schemaName, tableName, columnName, source) =>
     let
         OdbcSqlType.DATETIME = 9,
         OdbcSqlType.TYPE_DATE = 91,
         OdbcSqlType.TIME = 10,
         OdbcSqlType.TYPE_TIME = 92,
         OdbcSqlType.TIMESTAMP = 11,
         OdbcSqlType.TYPE_TIMESTAMP = 93,
         FixDataType = (dataType) =>
             if dataType = OdbcSqlType.DATETIME then
                 OdbcSqlType.TYPE_DATE
             else if dataType = OdbcSqlType.TIME then
                 OdbcSqlType.TYPE_TIME
             else if dataType = OdbcSqlType.TIMESTAMP then
                 OdbcSqlType.TYPE_TIMESTAMP
             else
                 dataType,
         FixDataTypeName = (dataTypeName) =>
             if dataTypeName = "TIMESTAMP" then
                 "TYPE_TIMESTAMP"
             else if dataTypeName = "DATE" then
                 "TYPE_DATE"
             else if dataTypeName = "TIME" then
                 "TYPE_TIME"
             else
                 dataTypeName,
         Transform = Table.TransformColumns(source, { { "DATA_TYPE", FixDataType }, { "TYPE_NAME", FixDataTypeName } })
     in
         Transform

This function changes the DATETIME, TIME, and TIMESTAMP types to TYPE_DATE, TYPE_TIME, and TYPE_TIMESTAMP types respectively. It makes similar changes to the type names as well.

Function:
(Text, Text, Text, Text, Table) → Table		Null
SqlTables

Function that allows the user to modify the table returned by calling SQLTables. This can be used to make changes to various properties of the tables, such as the table names.

Example:

SQLTables = (catalogName, schemaName, tableName, tableType, source) =>
    let
        FixTableName = (tableName) =>
            tableName & "_TEST",
        Transform = Table.TransformColumns(source, { "TABLE_NAME", FixTableName })
    in
        Transform

This function appends "_TEST" to the end of every table name in the result table returned by SQLTables.

Function:
(Text, Text, Text, Text, Table) → Table		Null

Table of all implicit type conversions. For binary operations between Type1 and Type2, the result will be of type ResultType.

OnError

A function that specifies how to handle any errors that are encountered. This is usually used to display useful error messages for specific types of errors that can be encountered.

Example:

OnError = (errorRecord as record) =>
    let
        OdbcError = errorRecord[Detail][OdbcErrors]{0},
        OdbcErrorMessage = OdbcError[Message],
        OdbcErrorCode = OdbcError[NativeError],
        IsSSLError = encryptionEnabled and (OdbcErrorCode = 1001 or OdbcErrorCode = 1003)
    in
        if IsSSLError then
            error Extension.CredentialError(Credential.EncryptionNotSupported, errorRecord[Message])
       else
            error errorRecord
